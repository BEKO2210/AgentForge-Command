# AgentForge Command v1.0.0 — Production-Ready Agent Cockpit

The first production-grade release: a premium **local mission-control cockpit**
for orchestrating a swarm of Claude Code agents — honest, local-first, zero
telemetry, security-audited, and tested across three OSes and three Node
versions.

## Highlights

- 🎮 **Premium cockpit** — Atlas Prime orchestrates; specialist cards, mascots,
  spend ledger, live workflow stepper. No mock activity, ever.
- 🌳 **Worktree isolation** — each specialist runs in its own `git worktree`
  on `agentforge/<id>` so parallel edits never collide.
- 💾 **Honest session reattach** — PTYs don't survive a restart; orphaned
  sessions are surfaced for one-click relaunch (no fake reattach).
- 🔒 **Security-hardened** — CSWSH closed (origin/host allowlist + per-session
  token), path-traversal hardening, schema-validated config, WS rate limits.
- ✅ **Proven quality** — Playwright E2E, axe a11y, a sub-millisecond matcher
  benchmark, c8 coverage threshold, and a cross-platform CI matrix.
- 🪶 **Dependency-light** — runtime is `ws` + `node-pty` (+ optional Rust
  accelerator); all MIT.
- 🧾 **Launch-ready docs** — threat model, privacy/DSGVO, trademark, known
  limits, third-party notices.

## Try it

```bash
# Local (test-harness mode — no API key needed)
npm install && npm start          # → http://localhost:4173

# Or containerised
docker compose up --build
```

See the [README](../../README.md) for live LLM and PTY modes, and the
[Policy notice](../../README.md) for PTY-vs-API guidance.

## Compatibility

- **Node.js:** 18 / 20 / 22 (CI-verified).
- **OS:** Linux (primary), macOS, Windows (build-verified; Docker/WSL
  recommended on Windows — see [`docs/INSTALL.md`](../INSTALL.md)).

## Honesty notes

- Coverage (82.84% lines) measures the **node-side** surface; the browser
  bundle is covered by E2E, not c8 (see [`KNOWN_LIMITS.md`](../../KNOWN_LIMITS.md)).
- Pro/Max subscriptions via third-party PTY frameworks are **not** permitted by
  Anthropic — use your own API key or the harness. See
  [`docs/THREAT_MODEL.md`](../THREAT_MODEL.md).
