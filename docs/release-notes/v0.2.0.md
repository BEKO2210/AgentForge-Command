# v0.2.0 — AgentForge Command (MIT)

The project re-frames itself as **AgentForge Command** — a local mission-control
cockpit for a Claude Code swarm led by Atlas Prime. The original 4-agent kit
(`.team/` scaffold, scripts, MCP server, 88-check test suite) is preserved
unchanged and lives on at `/console`; the new cockpit sits **on top of** it
as the default surface.

> *„Sag dem System einmal: du darfst Enter drücken — und es macht es ab jetzt
> für dich."* — Operator brief

## What's new

### Mission Control (`/`)

- **Atlas Prime + 11 specialists** with hand-built SVG mascots, role-specific
  briefings, channel callsigns, status pills with pulsing dots and per-agent
  terminals that breathe (blinking cursor, sweeping activity glow, fresh-line
  fade).
- **Atlas Command Panel** — hero panel with live mission stream (newest
  rule / spawn / broadcast events scroll in chronological order).
- **Spawn timeline** with vertical connector + colour-coded kinds + newest-
  event highlight.
- **Broadcast bar** — `/` focuses, `Enter` dispatches. Mock simulator by
  default; live LLM stream when `ANTHROPIC_API_KEY` is set.
- **Spawn-Builder modal** (`Alt+N`) for operator-defined specialists; new
  agents persist to `.team/arena.json` and survive restarts.
- **Detail drawer** with super-skill, briefing, capability chips, recent
  logs, lineage and per-agent controls.
- **Reduced motion** fully respected — kills animations, scanlines and glow.
- **Mobile**-aware grid (auto-fill 320px min), 44px tap targets, stacked
  hero on small screens.

### Persistent state

`.team/arena.json` (gitignored) holds:

- `evolution`     — mascot levels per agent
- `autoEnter`     — armed PTY ids
- `customAgents`  — operator-defined specialists
- `atlasMission`  — current mission goal

### Auto-enter watchdog

Server-side per-PTY toggle. Matches a conservative whitelist of permission
prompts (`(y/n)`, `press enter to continue`, `approve?`, `allow this tool
to run`, …) and presses Enter — single fire, 1.5s cooldown. Broadcasts an
`auto-fired` note back to the arena so the operator sees exactly what
happened.

### Per-specialist real Claude sessions

`gui/agents.json` ships a `specialists` array (Atlas + 11) with role-tuned
briefing prompts. They don't autostart; the operator launches each from the
arena WebSocket with `{t:"start-pty", id, goal}`. The server pastes the
matching prompt and presses Enter so the session boots straight into role.

### Live LLM bridge (optional)

`gui/llm.js` — zero npm deps. Uses `fetch` against the Anthropic Messages
API. Streams text deltas via SSE, reports token usage + cost. Falls back to
the mock simulator when no key is configured.

### Rust accelerator (optional)

`tools/forge-pulse/` — single-file, zero-dependency Rust binary. Pipes PTY
chunks → emits prompt + activity events. Advisory only — the JS matcher
drives auto-enter, so the system is unchanged when the binary is absent.
Crash-isolated; 5 unit tests; clippy-clean at `-D warnings`.

### Mascot pipeline

`scripts/render-mascots.mjs` renders every mascot from
`gui/public/arena/mascots.js` into standalone SVGs at `docs/mascots/*.svg`
plus a gallery. Same source for the live arena and the docs.

## Architecture changes

- `/` and `/arena` both serve Mission Control. `/console` serves the legacy
  surface. `/index.html` 0-second-redirects to `/`.
- New REST surface: `/api/agents`, `/api/state`, `/api/arena`.
- New WS protocol on `/arena` with messages: `auto-config`, `persist`,
  `press`, `input`, `start-pty`, `stop-pty`, `atlas-brief`,
  `atlas-brief-abort`.

## Upgrade notes

- If you bookmarked `/` of the old console, you'll now land in Mission
  Control. Use `/console` for the legacy view.
- If you want to disable the Rust accelerator (or never built it), set
  `FORGE_PULSE=0` or simply don't run `cargo build` in
  `tools/forge-pulse/`. The system is unchanged without it.
- If you want live LLM briefings, set `ANTHROPIC_API_KEY` on the server
  before starting. Without it, the broadcast bar uses the local mock.
